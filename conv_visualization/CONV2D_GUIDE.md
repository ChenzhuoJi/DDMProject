# PyTorch Conv2d 参数详解

## 1. 参数总览

```python
torch.nn.Conv2d(
    in_channels,      # 输入通道数
    out_channels,     # 输出通道数（卷积核个数）
    kernel_size,      # 卷积核尺寸（整数或元组）
    stride=1,         # 步长
    padding=0,        # 零填充
    dilation=1,       # 膨胀率
    groups=1,         # 分组数
    bias=True         # 偏置（本文不讨论）
)
```

---

## 2. 核心公式

### 输出尺寸公式

```
H_out = ⌊(H_in + 2P − D·(K−1) − 1) / S⌋ + 1
W_out = ⌊(W_in + 2P − D·(K−1) − 1) / S⌋ + 1
```

其中：
| 符号 | 参数 | 含义 |
|------|------|------|
| H_in | — | 输入高 |
| K | kernel_size | 卷积核尺寸 |
| S | stride | 步长 |
| P | padding | 填充数 |
| D | dilation | 膨胀率 |

### 参数量公式（无 bias）

```
Params = out_channels × (in_channels / groups) × K × K
```

## 3. 各参数详解

### 3.1 kernel_size (K)

卷积核的**空间尺寸**。

- **越大 → 感受野越大**，能捕获更广范围的局部特征
- **越大 → 参数量平方增长**（K² 关系）
- **越大 → 输出尺寸越小**（相同 padding 下）
- 常见选择：3×3（主流）、5×5、7×7；1×1 用于通道混合

**感受野 = K × K**（D=1 时）；大核可被小核堆叠替代（如两个 3×3 等效一个 5×5 的感受野，但参数量更少）。

### 3.2 stride (S)

卷积核每次滑动的**步长**。

- **S=1**：保留大部分空间信息，可能 same 卷积
- **S=2**：输出尺寸约减半（下采样 / 降分辨率）
- **S > 1 时，padding 无法完全补偿** — `H_out ≈ H_in / S`，这是固有下采样
- 常用于替代 MaxPooling 进行下采样

### 3.3 padding (P)

输入边界外**补零的层数**。

- 作用是**保持边界信息不被丢弃**，并控制输出尺寸
- **P 必须是整数**，因此 `P = D·(K−1)/2` 为整数时才能做到 same 卷积

> ⚠️ **常见误区**：以为 padding 能完全恢复输出尺寸。实际上 stride > 1 时 padding 无效；偶数 kernel 无法 exact same。

**常见 same 配置（K 奇数，S=1）：**

| K | D=1 | D=2 |
|---|-----|-----|
| 1 | P=0 | P=1 |
| 3 | P=1 | P=2 |
| 5 | P=2 | P=4 |
| 7 | P=3 | P=6 |

### 3.4 dilation (D)

卷积核各元素之间的**空洞间距**。

- **D=1**：标准卷积，元素紧邻
- **D>1**：空洞卷积（Atrous/Dilated Convolution）
- **有效核尺寸 = (K−1)×D + 1**，即用更大的视野但不增加参数
- **输出尺寸变小**（公式中 D·(K−1) 项增大）

```
K=3, D=1（标准）:         K=3, D=2（空洞）:
    ■ ■ ■                   ■   ■   ■
    ■ ■ ■                     ■   ■   ■
    ■ ■ ■                   ■   ■   ■
    覆盖 3×3 区域           覆盖 5×5 区域
    9个参数                 9个参数（不变！）
```

- 常用于语义分割（DeepLab 系列）：大感受野 + 高分辨率输出
- dilation > 1 时通常配合更大的 padding 使用

### 3.5 groups (G)

将输入/输出通道**分组卷积**。

- **G=1**：标准卷积，每个输出通道连接**所有**输入通道
- **G=C_in=C_out**：深度可分离卷积（Depthwise Conv），每个输入通道独立卷积
- **G 必须整除 C_in 和 C_out**

```
G=1（标准）:                  G=2（分组）:
Input:  4 channels           Input:  4 channels
Output: 4 channels           Output: 4 channels
                             ┌─────┬─────┐
每个输出连所有4个输入          │Grp1 │Grp2 │
Params: 4×4×K×K              │2→2  │2→2  │
                              └─────┴─────┘
                             Params: 4×2×K×K（减半！）
```

**分组卷积的作用：**
- **减少参数量**：参数量除以 G
- **减少计算量**：同等比例减少
- **通道间解耦**：每组独立学习特征
- 用于 MobileNet（Depthwise + Pointwise）、ResNeXt、ShuffleNet 等轻量网络

## 4. 不显而易见的规律

### 4.1 偶数 Kernel 无法 exact same

当 K 为偶数时，`D·(K−1)/2` 为半整数，无法用整数 P 实现 exact same（S=1 时）。例如 K=2, D=1 需要 P=0.5。因此**主流卷积核几乎全是奇数**（1, 3, 5, 7）。

### 4.2 Stride 决定下采样倍数

无论怎么调 padding 和 dilation，stride=S 时输出约是输入的 1/S。这是**硬约束**。想要 same shape 必须 S=1。

### 4.3 Dilation × Kernel 的等效关系

- D=2, K=3 等效于 D=1, K=5 的**感受野**（5×5），但参数只有 9 个而非 25 个
- 大 dilation 配合小 kernel 是"用参数换视野"
- D=2, K=3, S=1, P=2 → 输出尺寸与输入**相同**，感受野相当于 5×5

### 4.4 Groups 改变的是"连接密度"而非"空间卷积"

- Groups 不影响输出的空间尺寸
- Groups 只影响**通道间连接方式**和**参数量**
- G=C_in=C_out 时 = Depthwise Conv，每个通道独立学空间特征

### 4.5 多个参数的协同效应

这些参数不是独立作用的 —— 最终的输出尺寸由四个参数（K, S, P, D）联合决定。改变一个参数时，往往需要配合调整另一参数才能维持输出尺寸不变。

### 4.6 视觉任务中的典型组合

| 场景 | K | S | P | D | G |
|------|---|---|---|---|---|
| 标准特征提取 | 3 | 1 | 1 | 1 | 1 |
| 下采样 | 3 | 2 | 1 | 1 | 1 |
| 空洞卷积（分割） | 3 | 1 | 2 | 2 | 1 |
| 1×1 通道变换 | 1 | 1 | 0 | 1 | 1 |
| 深度可分离 | 3 | 1 | 1 | 1 | C_in=C_out |
| 分组轻量 | 3 | 1 | 1 | 1 | 2/4/32 |

---

## 5. 一句话总结

> **K 管视野，S 管缩放，P 管补偿，D 管空洞，G 管分组；输出尺寸由四者联合决定，S>1 时必下采样，偶数 K 无法 exact same。**
