# 需求文档：MDLM 实验复现与课程报告

## 项目背景

Spring 2026《随机过程》课程项目——离散扩散语言模型文献调研。
目标：复现 MDLM（Masked Diffusion Language Model）论文实验，产出报告所需的实验数据与可视化结果。

---

## 核心需求

### 1. 生成效果可视化（最高优先级）

**目标**：直观展示扩散模型从全 `[MASK]` 到完整句子的去噪过程，与自回归模型的逐个生成形成对比。

**交付物**：
- 多组 seed 下 `t=1.0 → t=0.0` 的逐行快照文件
- 不同采样步数（128 / 500 / 1000）的对比
- 与 AR 模型生成过程的概念对比

**成功标准**：文本可读、符合英文语法，如作者 Colab 所示：
> "do something else and then just work on yourself. You give them a lot of time..."

### 2. 采样器效率对比

**目标**：对比三种采样器的速度与质量。

**交付物**：
- ddpm_cache / ddpm / analytic 三者的耗时表
- Generative PPL 对比

### 3. text8 小规模训练

**目标**：从零训练一个 tiny diffusion 模型，展示训练收敛过程。

**交付物**：
- Loss 收敛曲线（val/nll vs step）
- 训练后的 checkpoint
- 从训练模型生成样本

## 技术约束

| 约束 | 说明 |
|:----|------|
| GPU | RTX PRO 6000（Blackwell, 102GB） |
| 软件环境 | PyTorch 2.7 + CUDA 12.8 + Python 3.12（容器预装） |
| 网络 | **不能直连 huggingface.co**，需使用镜像 `hf-mirror.com` |
| flash_attn | 官方 flash-attn 在 Blackwell 上编译困难，尽量避开硬依赖 |
| 代码 | 基于 `github.com/kuleshov-group/mdlm` 修改 |

### 已发现的坑 & 解决方案

| 问题 | 解决方案 |
|:----|---------|
| PyTorch 2.7 `weights_only=True` 阻止旧 checkpoint 加载 | `torch.load(..., weights_only=False)` 直接加载 |
| huggingface.co 不可达 | 设置 `HF_ENDPOINT=https://hf-mirror.com` |
| flash_attn 无法安装/编译 | 使用 `kuleshov-group/mdlm-no_flashattn-fp32-owt` 模型（fp32 版，不依赖 flash_attn） |
| Lightning `num_workers` 过多导致 ancdata 错误 | 设置 `loader.num_workers=8` |
| `val_check_interval` 超过 batch 数 | `trainer.val_check_interval=500` |
