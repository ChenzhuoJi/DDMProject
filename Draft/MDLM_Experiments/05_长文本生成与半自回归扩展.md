# 5. 长文本生成与半自回归扩展

预训练模型 `mdlm-no_flashattn-fp32-owt` 的最大上下文长度为 1024 token，这一限制源于模型训练时的序列长度设定。然而在实际应用中，生成更长文本的需求普遍存在。MDLM 框架提供了一种半自回归（Semi-Autoregressive, Semi-AR）扩展机制，通过 stride 方式逐步扩展上下文，在不重新训练模型的前提下突破长度限制。

半自回归扩展的核心思路可以概括为"分步生成、拼接迭代"。首先生成一段长度为 `L`（= 1024）的标准文本，随后将末端的 `S` 个 token（stride length）作为新生成段的上下文条件，拼接后继续向后生成。每次迭代新增 `S` 个 token，经 `N` 次迭代后总长度可达 `L + N × S`。这一机制与纯自回归模型的核心区别在于：每个 stride 段的内部生成仍然是全局并行的扩散过程，而非逐 token 递推，因此它在扩展长度的同时保留了扩散模型的双向上下文建模优势。

我们在 seed 123 下采用 Semi-AR 模式生成了长度为 1024 token 的长文本，stride length 设为 512，分两段扩展到 2048 token 的可用上下文。模型生成了约 1024 个 token 的连贯文本，内容呈现政治与社会评论风格，其中包含对奥巴马时代跨性别厕所法案的讨论、奥兰多夜店枪击事件报道等话题。模型在句子层面保持了较好的连贯性，能够生成包含直接引语与复合句的多段落结构。整段文本在话题转换处偶见逻辑断裂——例如从厕所法案争议突然跳转到奥兰多枪击事件——这是无条件生成模式下模型缺乏全局主题控制的固有局限，并非半自回归扩展机制本身的问题。

在耗时方面，1024 token 长度的完整生成为 87.3 秒，PPL 为 43.8。与 256 token 长度（seed 123，9.1 秒，PPL 41.2）相比，生成长度扩展至 4 倍，耗时增加约 9.6 倍，大致呈超线性关系。这一现象的原因在于两个方面：其一，ddpm_cache 采样器的缓存机制在较长序列中的命中率有所下降（更多 token 同步变化）；其二，自注意力机制的计算复杂度随序列长度呈二次增长。在 Colab T4 平台上，1024 token 的一次生成可在 90 秒内完成，尚处于可接受的范围。如需更高效的长文本生产，可考虑降低采样步数（如从 1000 步降至 500 步）或采用更长的 stride length 以减少分段次数。
