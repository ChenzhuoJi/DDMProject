# 3. 采样算法效率分析

离散扩散模型的推理效率在很大程度上取决于反向采样策略的选择。MDLM 框架提供了三种不同的采样算法——ddpm_cache、ddpm 与 analytic——它们在每步的计算成本与网络调用频率上存在显著差异。本节在固定随机种子（seed 42）、采样步数（1000）与序列长度（256 token）的条件下，对三种采样器进行了系统性的速度与质量对比评测。

## 3.1 三种采样器的设计原理

ddpm 采样器（标准祖先采样）的实现最为直接：在每个离散时间步，计算当前噪声水平下的网络输出 `p_θ(x₀|xₜ)`，据此采样得到下一步状态 `xₜ₋₁`，共计调用网络 1000 次。这一过程完整地复现了马尔可夫链的每一步转移，但计算开销与步数呈严格线性关系。

ddpm_cache 采样器在此基础上引入了一项关键优化：缓存上一步计算的 `p_θ(x₀|xₜ)`，并在当前步中比较新采样的 `xₜ₋₁` 与旧的 `xₜ` 是否一致。若序列中所有 token 均未发生变化（即模型判定当前状态已足够确定，无需进一步更新），则跳过本轮网络前向计算，直接复用缓存值。这一优化之所以成立，根本原因在于扩散过程满足马尔可夫性（无后效性）：给定当前状态 `xₜ`，下一步状态 `xₜ₋₁` 的条件分布完全由 `p_θ(x₀|xₜ)` 与当前时间步决定，与更早的历史状态无关。因此，当 `xₜ` 相对于上一步未发生变化时，缓存的条件概率分布仍然有效。

analytic 采样器采用基于得分函数的解析更新策略，每步均需计算完整的得分矩阵并执行矩阵指数运算，计算量在三者中最大。

## 3.2 速度与质量评测结果

三种采样器的对比结果在图 3 中可视化呈现。ddpm_cache 以 9.6 秒的端到端耗时位居最快，ddpm（31.8 秒）耗时为其 3.30 倍，analytic（36.5 秒）则达到 3.79 倍。ddpm_cache 相对于 ddpm 的加速效果（3.3×）显著高于先前研究报道的 2–3 倍，其原因在于本实验采用的序列长度较短（256 token），在较短序列中 token 不发生变化的比例更高，缓存的命中率也随之提升。

*图 3：ddpm_cache、ddpm、analytic 三种采样器的端到端耗时对比（seed=42, steps=1000, length=256）。柱顶标注为绝对耗时(s)，柱内标注为相对 ddpm_cache 的倍率。脚注说明三者 PPL（52.5） 以及生成的文本完全一致。*

在生成质量方面，三者的 Generative PPL 完全一致（均为 52.5），经人工核查，所生成文本逐 token 相同。这一等价性在理论上是预期之内的：在相同随机种子控制下，三种采样器共享同一初始噪声状态与同一组模型参数，它们的差异仅在于条件概率分布的数值计算路径不同，而最终采样结果由同一组 Gumbel 噪声与模型 logits 共同决定，不应出现分歧。analytic 采样器虽在数学推导上采用了不同的更新公式，但在 ddpm 与 ddpm_cache 所使用的祖先采样框架下，三者的每一步状态更新均可化为同等形式，因此等价性成立。

值得注意的是，analytic 采样器虽在耗时上最为昂贵，但在某些噪声条件下可能具有更优的数值稳定性。本实验中三种采样器表现一致，从一个侧面验证了 MDLM 框架下实现的一致性——用户可根据计算资源与延迟要求自由选择采样策略，而无需担心引入系统性的质量偏差。
